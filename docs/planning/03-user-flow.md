# User Flow

## 1. 메인 플로우: 방문자 → 문의

```
┌─────────────────────────────────────────────────────────────────┐
│                    방문자 여정 (User Journey)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [유입]                                                          │
│    │                                                             │
│    ├─ 검색 (SEO)                                                 │
│    ├─ 직접 URL 입력                                               │
│    └─ 링크 공유 (SNS, 이메일)                                     │
│    │                                                             │
│    ▼                                                             │
│  [히어로 섹션] ─────────────────────────────────────────────────  │
│    │ 3초 내 "어떤 회사인지" 파악                                  │
│    │ 관심 있으면 CTA 클릭                                         │
│    │                                                             │
│    ▼                                                             │
│  [서비스 섹션] ─────────────────────────────────────────────────  │
│    │ "무엇을 하는 회사인지" 상세 확인                             │
│    │ AI 컨설팅 / 시스템 개발                                     │
│    │                                                             │
│    ▼                                                             │
│  [기술 스택 섹션] ──────────────────────────────────────────────  │
│    │ "어떤 기술을 쓰는지" 확인                                    │
│    │ 전문성 판단                                                  │
│    │                                                             │
│    ▼                                                             │
│  [회사 소개 섹션] ──────────────────────────────────────────────  │
│    │ "누가 운영하는지" 확인                                       │
│    │ 신뢰감 형성                                                  │
│    │                                                             │
│    ▼                                                             │
│  [연락처 섹션] ─────────────────────────────────────────────────  │
│    │ 문의 결정                                                    │
│    │                                                             │
│    ▼                                                             │
│  [이메일 발송] ─────────────────────────────────────────────────  │
│    │                                                             │
│    ▼                                                             │
│  [목표 달성: 문의 완료]                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 페이지 내 네비게이션

### 2.1 헤더 네비게이션

```
[로고] ──────── [회사소개] [서비스] [기술스택] [연락처]
                    │          │        │         │
                    ▼          ▼        ▼         ▼
               #about    #services  #tech    #contact
```

### 2.2 스크롤 기반 이동

```
┌────────────────────────┐
│ Hero (뷰포트 100%)      │ ← 첫 화면
│ CTA: "서비스 보기" ──────┼───────┐
└────────────────────────┘       │
                                  │
┌────────────────────────┐       │
│ Services              │ ◄──────┘
│                        │
└────────────────────────┘
            │
            ▼ (자연스러운 스크롤)
┌────────────────────────┐
│ Tech Stack             │
└────────────────────────┘
            │
            ▼
┌────────────────────────┐
│ About                  │
└────────────────────────┘
            │
            ▼
┌────────────────────────┐
│ Portfolio (준비 중)     │
└────────────────────────┘
            │
            ▼
┌────────────────────────┐
│ Contact                │
│ [이메일 링크]           │ ← mailto: 링크
└────────────────────────┘
```

## 3. 인터랙션 정의

### 3.1 CTA 버튼
| 요소 | 액션 | 결과 |
|------|------|------|
| 히어로 CTA | 클릭 | 서비스 섹션으로 스무스 스크롤 |
| 헤더 메뉴 | 클릭 | 해당 섹션으로 스무스 스크롤 |
| 이메일 링크 | 클릭 | 이메일 앱 실행 (mailto:) |
| 로고 | 클릭 | 페이지 최상단으로 이동 |

### 3.2 스크롤 동작
| 동작 | 반응 |
|------|------|
| 스크롤 다운 | 헤더 고정 (sticky) |
| 스크롤 업 | 헤더 표시 |
| 섹션 진입 | 해당 메뉴 활성화 (선택) |

## 4. 모바일 플로우

```
┌─────────────────────┐
│ [≡] 로고             │  ← 햄버거 메뉴
└─────────────────────┘
         │
         ▼ 클릭
┌─────────────────────┐
│ 회사소개             │
│ 서비스               │
│ 기술스택             │
│ 연락처               │
└─────────────────────┘
         │
         ▼ 메뉴 선택
┌─────────────────────┐
│ 해당 섹션으로 이동   │
│ 메뉴 자동 닫힘       │
└─────────────────────┘
```

## 5. 이탈 포인트 분석

| 이탈 포인트 | 예상 원인 | 대응 방안 |
|------------|----------|----------|
| 히어로에서 | 관심 없음 / 타겟 불일치 | tagline 개선, CTA 명확화 |
| 서비스에서 | 원하는 서비스 아님 | 서비스 설명 구체화 |
| 기술스택에서 | 기술 불일치 | (타겟 정확도 문제) |
| 연락처 전 | 신뢰 부족 | 포트폴리오 추가 (추후) |
