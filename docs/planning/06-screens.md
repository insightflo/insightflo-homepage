# Screens (화면 명세)

## 1. 페이지 구조 개요

**타입**: 원페이지 (Single Page Application with Sections)
**URL**: `/` (index)

```
┌─────────────────────────────────────────┐
│ Header (고정)                            │
├─────────────────────────────────────────┤
│ Hero Section (#hero)                    │
├─────────────────────────────────────────┤
│ Services Section (#services)            │
├─────────────────────────────────────────┤
│ Tech Stack Section (#tech)              │
├─────────────────────────────────────────┤
│ About Section (#about)                  │
├─────────────────────────────────────────┤
│ Portfolio Section (#portfolio) [준비중]  │
├─────────────────────────────────────────┤
│ Contact Section (#contact)              │
├─────────────────────────────────────────┤
│ Footer                                  │
└─────────────────────────────────────────┘
```

---

## 2. 섹션별 상세 명세

### 2.1 Header (고정 헤더)

**위치**: 최상단, sticky
**진입점**: 항상 표시

| 요소 | 설명 | 액션 |
|------|------|------|
| 로고 | 회사명/로고 이미지 | 클릭 → 페이지 최상단 |
| 네비게이션 | 회사소개, 서비스, 기술스택, 연락처 | 클릭 → 해당 섹션 스크롤 |
| 모바일 메뉴 | 햄버거 아이콘 (모바일) | 클릭 → 메뉴 토글 |

**데이터**:
- 네비게이션 항목 (하드코딩)

**반응형**:
- Desktop: 가로 메뉴
- Mobile: 햄버거 메뉴

---

### 2.2 Hero Section (#hero)

**목적**: 첫인상, 3초 내 회사 정체성 전달
**진입점**: 페이지 최초 로드

| 요소 | 설명 | 스타일 |
|------|------|--------|
| 메인 타이틀 | 회사명 | H1, 굵게 |
| 서브 타이틀 | 한 줄 소개 (문제해결 강조) | H2 또는 큰 본문 |
| CTA 버튼 | "서비스 보기" | Primary Button |
| 배경 | 그라데이션 또는 이미지 | 딥블루 계열 |

**데이터**:
- 회사명 (하드코딩)
- 슬로건/tagline (하드코딩)

**인터랙션**:
- CTA 클릭 → Services 섹션으로 스무스 스크롤

---

### 2.3 Services Section (#services)

**목적**: 제공 서비스 안내
**진입점**: 히어로 CTA, 헤더 메뉴

| 요소 | 설명 |
|------|------|
| 섹션 타이틀 | "서비스" |
| 서비스 카드 1 | AI 컨설팅 |
| 서비스 카드 2 | 시스템 개발 |

**서비스 카드 구성**:
| 요소 | 설명 |
|------|------|
| 아이콘 | 서비스를 나타내는 아이콘 |
| 제목 | 서비스명 |
| 설명 | 2-3줄 설명 |
| (선택) 특징 | 불릿 포인트 |

**데이터**:
```typescript
interface Service {
  icon: string;
  title: string;
  description: string;
  features?: string[];
}
```

**레이아웃**:
- Desktop: 2열 그리드
- Mobile: 1열 스택

---

### 2.4 Tech Stack Section (#tech)

**목적**: 기술 역량 어필
**진입점**: 헤더 메뉴, 스크롤

| 요소 | 설명 |
|------|------|
| 섹션 타이틀 | "기술 스택" |
| 기술 뱃지 | 아이콘 + 이름 |

**기술 목록**:
| 카테고리 | 기술 |
|----------|------|
| Frontend | Vue |
| Backend | Java, Express.js |
| AI | Claude Code, OCR, Automation |
| Cloud | AWS, GCP |
| Database | Oracle, PostgreSQL |
| RAG | Langchain, LangGraph, VectorDB |

**레이아웃**:
- 그리드 (6열 Desktop, 3열 Tablet, 2열 Mobile)
- 또는 카테고리별 그룹핑

**데이터**:
```typescript
interface TechItem {
  name: string;
  icon: string;
  category: string;
}
```

---

### 2.5 About Section (#about)

**목적**: 회사/대표 소개, 신뢰감 형성
**진입점**: 헤더 메뉴, 스크롤

| 요소 | 설명 |
|------|------|
| 섹션 타이틀 | "회사 소개" |
| 비전/미션 | 한 줄 또는 짧은 문단 |
| 가치관 | 핵심 가치 2-3개 |
| 대표 소개 | (선택) 간략한 소개 |

**레이아웃**:
- 텍스트 중심 레이아웃
- 또는 좌우 분할 (이미지 + 텍스트)

---

### 2.6 Portfolio Section (#portfolio) - 준비 중

**목적**: 프로젝트 사례로 역량 증명
**현재 상태**: 준비 중 표시

| 요소 | 설명 |
|------|------|
| 섹션 타이틀 | "포트폴리오" |
| 상태 메시지 | "준비 중입니다" 또는 "Coming Soon" |

**향후 계획**:
- 프로젝트 카드 그리드
- 필터 (AI, Web, App, SI)
- 상세 모달 또는 별도 페이지

---

### 2.7 Contact Section (#contact)

**목적**: 문의 유도
**진입점**: 헤더 메뉴, 스크롤

| 요소 | 설명 |
|------|------|
| 섹션 타이틀 | "문의하기" |
| 안내 문구 | "프로젝트 문의는 이메일로 연락주세요" |
| 이메일 | 이메일 주소 (mailto: 링크) |
| CTA 버튼 | "이메일 보내기" (선택) |

**인터랙션**:
- 이메일 클릭 → 이메일 앱 실행

---

### 2.8 Footer

**목적**: 저작권, 부가 정보
**위치**: 최하단

| 요소 | 설명 |
|------|------|
| 저작권 | "© 2025 회사명. All rights reserved." |
| (선택) SNS 링크 | GitHub, LinkedIn 등 |

---

## 3. 화면 간 연결 맵

```
Header [항상 표시]
  │
  ├─ 로고 ──────────────────────→ #hero (최상단)
  │
  ├─ 회사소개 ──────────────────→ #about
  │
  ├─ 서비스 ────────────────────→ #services
  │
  ├─ 기술스택 ──────────────────→ #tech
  │
  └─ 연락처 ────────────────────→ #contact

Hero Section
  │
  └─ CTA "서비스 보기" ─────────→ #services

Contact Section
  │
  └─ 이메일 링크 ────────────────→ mailto:email@domain.com
```
