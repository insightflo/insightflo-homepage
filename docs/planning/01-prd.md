# PRD (Product Requirements Document)

## MVP 캡슐

```
1. 목표: 프로젝트 의뢰 기업과 협업 파트너에게 신뢰감 형성
2. 페르소나: IT 프로젝트 의뢰 기업 담당자, 협업 파트너사, 일반 방문자
3. 핵심 기능: 회사 소개, 서비스 안내, 기술 역량 어필, 연락처 제공
4. 성공 지표 (노스스타): 문의 전환율 (페이지 방문 → 이메일 문의)
5. 입력 지표: 페이지 체류 시간, 스크롤 깊이
6. 비기능 요구: 모바일 반응형, 빠른 로딩 (Lighthouse 90+)
7. Out-of-scope: 블로그, 회원가입, 결제 기능, 다국어 지원 (1차)
8. Top 리스크: 포트폴리오 부재로 신뢰감 부족
9. 완화/실험: 기술 스택 + 경력으로 대체, 런칭 후 빠르게 포트폴리오 추가
10. 다음 단계: Astro 프로젝트 생성 및 기본 레이아웃 구현
```

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
개인사업자 회사 소개 페이지

### 1.2 프로젝트 배경
- 개인사업자로서 AI 활용 웹/앱 개발 및 SI/시스템 구축 사업 운영
- 프로젝트 의뢰 기업과 협업 파트너에게 전문성과 신뢰감을 전달할 온라인 창구 필요
- GitHub Pages를 활용한 정적 사이트로 비용 효율적 운영

### 1.3 목표
- **Primary**: 방문자에게 전문성과 신뢰감 형성
- **Secondary**: 서비스 안내를 통한 문의 유도

---

## 2. 타겟 사용자

### 2.1 Primary Persona: 프로젝트 의뢰 검토 기업
- **Who**: IT 프로젝트 외주를 검토하는 기업 담당자
- **Pain**: 믿을 수 있는 개발 파트너를 찾기 어려움
- **Want**: 기술력과 경험을 확인하고 싶음
- **Goal**: 문의 연락 → 상담

### 2.2 Secondary Persona: 협업 파트너사
- **Who**: 함께 프로젝트를 진행할 파트너사
- **Pain**: 협업 가능한 기술 스택과 역량 파악 필요
- **Want**: 보유 기술과 서비스 범위 확인
- **Goal**: 협업 제안 연락

### 2.3 Tertiary Persona: 일반 방문자
- **Who**: 검색이나 SNS를 통해 유입된 방문자
- **Pain**: 어떤 회사인지 빠르게 파악하고 싶음
- **Want**: 한눈에 이해되는 정보
- **Goal**: 정보 획득 후 이탈 또는 관심 시 문의

## 3. 핵심 기능

### 3.1 히어로 섹션
- **목적**: 첫인상, 3초 내 어떤 회사인지 전달
- **구성요소**:
  - 회사명/브랜드명
  - 한 줄 소개 (문제해결 강조)
  - CTA 버튼 (서비스 섹션으로 스크롤)

### 3.2 회사 소개 섹션
- **목적**: 신뢰감 형성, 비전과 가치관 전달
- **구성요소**:
  - 비전/미션 문구
  - 가치관
  - 대표 소개 (간략)

### 3.3 서비스 섹션
- **목적**: 제공 서비스 명확히 안내
- **구성요소**:
  - AI 컨설팅
  - 시스템 개발

### 3.4 기술 스택 섹션
- **목적**: 전문성 어필
- **구성요소**:
  - 기술 아이콘 나열
  - Frontend: Vue
  - Backend: Java, Express.js
  - AI: Claude Code, OCR, Automation
  - Cloud: AWS, GCP
  - Database: Oracle, PostgreSQL
  - RAG: Langchain, LangGraph, VectorDB

### 3.5 포트폴리오 섹션 (준비 중)
- **목적**: 실제 프로젝트 사례로 역량 증명
- **현재 상태**: 준비 중
- **계획**: 런칭 후 순차 추가

### 3.6 연락처 섹션
- **목적**: 문의 유도
- **구성요소**:
  - 이메일 주소
  - (선택) 간단한 문의 안내 문구

## 5. 벤치마킹 분석

### 참고 원칙 (출처: [Shopify](https://www.shopify.com/kr/blog/how-to-write-an-about-us-page))
1. **강력한 브랜드 아이덴티티**: 일관된 색상, 폰트, 이미지
2. **사용자 친화적 디자인**: 명확한 탐색, 간단한 레이아웃
3. **고품질 시각 콘텐츠**: 고화질 이미지 사용
4. **"말하지 말고 보여주기"**: 가치 제안을 시각적으로 표현

### 차별화 포인트
- AI 기술 (Claude, RAG, LangChain) 활용 역량 강조
- 풀스택 역량 (Frontend + Backend + AI + Cloud)
- 개인사업자의 유연성과 전문성 조합

## 6. 가설 vs 사실

| 항목 | 가설 | 사실 | 검증 방법 |
|------|------|------|----------|
| 경력/기술 정보가 신뢰감을 준다 | O | - | 런칭 후 피드백 수집 |
| 포트폴리오 없이도 기술 스택으로 신뢰 가능 | O | - | 문의 전환율 측정 |
| 원페이지가 정보 전달에 효과적 | O | - | 스크롤 깊이 분석 |

| 연락 폼 | 이메일만 | 정적 사이트 제약 | Formspree 등 추후 검토 |

## 8. 실패 대응

- **MVP 정의**: 6개 섹션 원페이지, 포트폴리오 제외
- **실패 시나리오**: 문의가 오지 않음
- **학습 목표**: 어떤 정보가 부족한지 피드백 수집
- **Pivot 옵션**:
  - 포트폴리오 빠르게 추가
  - 블로그/콘텐츠 마케팅 병행
  - 다른 채널(LinkedIn 등) 연계
